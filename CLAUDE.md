# CLAUDE.md

## AI rules

This repository's coding rules live in `docs/ai-rules/` — one directory per category, one
rule per `.md` file, each with Good/Bad examples taken from this repo. The project concept
(what the app is, who it's for, what to optimize for) is in
`docs/ai-rules/concept/project-brief.md`.

Categories: `general` (FDD architecture, boundaries, aliases, thin routes), `typescript`,
`svelte`, `styling`, `api-design`, `state`, `i18n`, `testing`, `git`.

Before writing or modifying code, consult the rule files of the relevant categories and
follow them. `docs/ai-rules/ROADMAP.md` is the living knowledge snapshot of the stack and
rule coverage.
